# Feedback & Collaboration

We're actively seeking feedback from testers! Here's how to participate.

## Round 1 Testing (Detailed)

**Assigned a module?** Use the [Round 1 Feedback Form](https://docs.google.com/forms/d/e/ROUND1_FORM_ID/viewform) for detailed module-specific feedback.

Modules:
- **Module A** — Install Test
- **Module C** — Docs Clarity Review
- **Module G** — Video Walkthrough Feedback

See [docs/ROUND1_FEEDBACK_FORM.md](docs/ROUND1_FEEDBACK_FORM.md) for form structure.

## Quick Feedback (2 min)

**Just exploring?** Use our [Quick Feedback Form](https://docs.google.com/forms/d/e/1FAIpQLSeKLGv49IieXanF3zVxE5YwB_I6mLxANe3NP6_ApsY1XR1-eA/viewform) - simple questions, no GitHub account needed.

## Discussion Categories

- **Q&A** - Questions about using the ADK
- **Ideas** - Feature requests and suggestions
- **Show & Tell** - Share what you built
- **Setup Problems** - Installation and config help
- **Docs Confusion** - Where docs need improvement

## Google Form Questions

*For setting up the intake form:*

### Required
1. **Which link did you start with?**
   - README quickstart
   - Documentation site
   - Example code
   - Other (please specify)

2. **What module did you test?**
   - Fluent API (fluent.py)
   - Validation (validator.py)
   - MCP Server
   - Client examples (Go/Rust/TypeScript)
   - Full pipeline
   - Just exploring

3. **First confusing step** *(text)*
   > Where did you first get stuck or confused?

4. **First error message** *(text, optional)*
   > Copy/paste any error you saw

5. **Did you use AI assistance?**
   - Yes (ChatGPT, Claude, Copilot, etc.)
   - No

### Optional
6. **What would have helped?** *(text)*
   > Better docs? Video? Different examples?

7. **Overall experience** *(1-5 scale)*
   > 1 = Very frustrating, 5 = Smooth

8. **Final recommendation** *(text)*
   > Any other feedback for the team?

9. **Email** *(optional)*
   > Only if you'd like a response
